## 4. 一键部署默认场景

点击左侧“一键部署”或“加载场景”按钮，系统会部署默认演示场景：

- 我方：一个 AI 士兵班组，作为整体单元 `blue_squad_1`。
- 敌方：一个敌方火力点 `enemy_1`。
- 风险区：大路狙击/暴露风险区。
- 目标：目标建筑入口 `target_entry_1`。
- 地图：从 `wargame` 复制到本仓库的 `roads.geojson`、`buildings.geojson`、`water.geojson`、`landhouse.geojson`。

## 5. 拖拽部署敌我与威胁区

地图上的我方班组、敌方单元和目标入口为可拖拽 marker。拖拽后，左侧对象列表会用 VBS 本地坐标显示新位置。

当前 v0 的拖拽主要用于演示与人工调整，不会把拖拽坐标持久化回后端数据库；后续如需要闭环保存，可增加 `POST /api/demo-scene/update-object`。

## 6. 演示命令一：默认路径规划

加载默认场景后，系统会先运行确定性寻路算法，生成 Top-N 候选路径。默认情况下，路径规划器会输出至少两条路线：

- `route_main`：大路，距离短，但有较高风险。
- `route_side`：小路，距离略长，但风险更低。

右侧候选路线区会显示距离、风险、总分和标签；地图中会同时绘制候选路线，并高亮当前推荐路线。

## 7. 演示命令二：大路狙击风险导致重规划

在右侧聊天框输入：

```text
不要走大路，大路有狙击风险
```

系统执行流程：

1. 固定寻路算法已经先生成候选路线。
2. LLM 约束解析门面把用户输入解析为结构化补丁：`avoid road_class:main_road`。
3. 寻路算法应用补丁重新评分。
4. 推荐路线从 `route_main` 切换为 `route_side`。
5. 可视化路线、候选列表、约束列表和 Adapter 预览同步更新。

## 8. 演示命令三：忽略敌方营地并争分夺秒

点击“重置会话”后输入：

```text
敌军不在营地，现在必须争分夺秒
```

系统执行流程：

1. LLM 约束解析门面把输入解析为 `ignore_zone enemy_zone:enemy_1`，并设置 `priority=time`。
2. 路径规划降低敌方区域对路径评分的影响。
3. 推荐路线回到更短的 `route_main`。

该演示对应用户提出的 case 2：算法本能规避敌方区域，但用户给出新的战术假设后，LLM 只修改约束，不直接生成路径坐标。

## 9. 候选路径接近时如何选择

当 `scores_are_close()` 判断多条候选路线分数接近时，系统不会直接替用户拍板，而会调用候选路线解释函数，说明每条路径的距离、风险、总分和标签，提示用户选择 `1`、`2`、`3`，或输入新的约束重新规划。

当前 v0 已具备解释文本和候选列表展示；编号选择持久化可作为后续增强。

## 10. 会话恢复与重置

右侧聊天栏保留轻量 SQLite 会话记忆：

- 用户输入会记录为 message。
- LLM 解析出的约束补丁会记录为 constraints。
- 候选路径和推荐路径会记录到当前 session。

点击“重置会话”会清空当前 session 的历史消息和约束。v0 的会话记忆用于演示，不承诺生产级持久化。

## 11. Adapter 预览如何理解

Adapter 预览展示的是上层准备发给下层仿真系统的结构化意图：actor、waypoints、约束和当前选择路径。当前本机没有真实 VBS 引擎，所以下层提交使用 mock/预览方式。

真实部署时，下层仿真系统需要按 `zhibing/docs/lower_simulation_interface.md` 提供 HTTP 或 socket 接口。默认推荐 HTTP；配置中也预留 socket 长连接。
