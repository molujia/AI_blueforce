# 智兵决策 v0 演示闭环更新设计

## 目标

本轮更新把现有“智兵决策”系统修正为一个可演示、可测试、边界清楚的 v0 指挥闭环。系统必须能够在用户进入前完成 GraphRAG 知识库准备，在用户进入后显示参考 `C:\Users\22646\Desktop\wargame` 的 2D 战术地图界面，并用“确定性寻路算法 + LLM 特殊约束处理 + 轻量会话记忆”完成从自然语言意图到路径选择、可视化展示、下层仿真 Adapter 预览的一致链路。

## 范围

纳入本轮：

- 复用 wargame 的 2D 地图资产和战术 HUD 视觉布局。
- 一个我方 AI 士兵班组整体单元，一个或多个敌方单元/火力点，目标建筑入口，威胁区/狙击风险区。
- 一键部署默认 demo，也允许在地图上拖拽部署敌我、目标和威胁区。
- Top-N 候选路径规划，默认 `N=3`。
- LLM 解析用户自然语言特殊要求，并输出结构化约束补丁。
- 候选路径分数接近时，由 LLM 解释路线优缺点并等待用户选择。
- 轻量 SQL 会话记忆，保存聊天历史、当前有效约束、路径候选、用户选择和解释。
- GraphRAG 可用性验收：官方 quickstart 示例可跑通，用户提供的 PDF/DOCX 能建库、查询并返回来源片段。
- 文档补充启动、前置知识库构建、场景部署、演示流程和测试命令。

不纳入本轮：

- 3D 地球开场和真实地图/卫星图切换。
- wargame 的四类士兵体系、装备库、弹药油料、保障脱链等复杂机制。
- 让 LLM 直接生成最终路线坐标。
- 评估 GraphRAG 抽取出的军事知识是否足够有效。本轮只验可用性。
- 生产级持久化、多用户权限、服务端高可用。

## 页面与交互设计

可视化中间层采用 wargame 风格的战术 HUD：

- 顶部标题栏显示系统名、场景名、仿真连接状态、GraphRAG 状态、LLM 模式、会话编号。
- 顶部 HUD 状态栏显示当前任务阶段、坐标、候选路径数量、是否等待用户选择、Adapter 预览状态。
- 中央是 2D Leaflet 地图，只加载本仓库复制的 GeoJSON 图层。
- 左侧边栏包含“一键部署”“兵力与威胁部署”“场景对象列表”。
- 右侧边栏保留聊天式交互，包含自然语言输入、历史消息、当前有效约束、候选路径说明、LLM 解释、重置会话按钮、Adapter 指令预览。

地图基础图层至少包含：

- 道路：来自 wargame 的 `roads.geojson` 或等价道路数据。
- 建筑：来自 wargame 的 `buildings.geojson`。
- 可选水系/地块：如 `water.geojson`、`landhouse.geojson`，按实现成本选择。

任务图层至少包含：

- 我方班组单元。
- 敌方单元/火力点。
- 目标建筑入口。
- 威胁区、敌方火力区、狙击风险区。
- Top-N 候选路径。
- 最终选择路径。
- 被约束排除的路线或路段。

## 场景模型

v0 演示场景使用一个简化但可审计的数据模型：

- `friendly_squad`：我方 AI 士兵班组整体单元，类似 wargame 中一个可拖拽单位。
- `enemy_unit`：敌方单元或火力点，带位置、半径、威胁类型。
- `risk_zone`：非敌军实体风险区，例如“大路狙击风险”。
- `target_entry`：目标建筑入口点。
- `route_candidate`：候选路径，包含 waypoint、距离、耗时、风险、约束命中情况和分数。
- `constraint_patch`：LLM 从用户输入解析出的结构化约束。

默认一键部署必须能创建一个足够简单的场景：一个我方班组、一个目标建筑入口、一条短而直的大路路径、一条绕行小路路径、一个可选敌方或风险区。该场景用于证明寻路有效、能考虑敌人、能被 LLM 特殊要求影响。

## 寻路与 LLM 协作

固定流程如下：

1. 用户提交任务意图或点击一键部署。
2. 确定性寻路算法先执行，生成 Top-N 候选路径。
3. 系统计算每条路径的距离、风险、耗时、约束惩罚和总分。
4. 如果最优路径与次优路径分差明显，系统直接推荐最优路径。
5. 如果 Top-N 路径分数接近，系统把候选路径摘要交给 LLM，由 LLM 解释每条路径优缺点，并等待用户选择 `1..N` 或输入“其它要求”。
6. 用户输入新要求时，LLM 不直接画路线，而是输出结构化 `constraint_patch`。
7. 系统把约束补丁加入当前会话有效约束集合，重新运行寻路算法。
8. 地图和右侧面板展示新路径、约束来源、路径改变原因。

示例约束补丁：

```json
{
  "constraint_id": "c_road_sniper_risk",
  "source_text": "不要执行，因为大路有被狙击的风险",
  "action": "avoid",
  "target_type": "road_class",
  "target_id": "main_road",
  "weight_delta": 80,
  "reason": "用户明确提出大路存在狙击风险"
}
```

```json
{
  "constraint_id": "c_ignore_enemy_camp",
  "source_text": "不要绕路，因为敌军不在营地，现在必须争分夺秒",
  "action": "ignore_zone",
  "target_type": "enemy_zone",
  "target_id": "enemy_1",
  "priority": "time",
  "reason": "用户说明该敌军区域不应影响本次路径"
}
```

GraphRAG 本轮只承担“前置知识库可用”和“运行时可查询”的职责。

验收必须包含：

- 跑通 Microsoft GraphRAG 官方 quickstart 或官方示例流程，包括初始化、索引和查询。
- 对 `C:\Users\22646\Desktop\各种测试\智兵决策\test_files\地下作战.docx` 建库并查询，返回命中来源片段。
- 对 `C:\Users\22646\Desktop\各种测试\智兵决策\test_files\ARN19656_ATP.pdf` 建库并查询，返回命中来源片段。
- 界面或命令行能显示知识库状态、索引文件位置、查询命中来源。

测试阶段使用火山模型 `ep-20260615114505-247zc` 时，必须先估计调用次数和成本，并征得用户同意。最终本地部署使用本地 LLM；如果要使用视觉模型，必须另行征得用户同意。

## 会话记忆

右侧聊天栏保留 wargame 的会话式指挥思路，但不原样迁移全量业务脚本。v0 使用轻量 SQL 演示层保存：

- `session_id`
- 场景快照
- 聊天消息
- 当前有效约束补丁
- 候选路径摘要
- 用户最终选择
- LLM 解释和追问

系统刷新或重启后可以恢复最近会话。用户可以点击“重置会话”，清空聊天历史、约束补丁、候选路径和选择记录，从零开始当前演示。v0 不要求生产级持久化，不要求多用户隔离。

## 下层仿真边界

本轮系统仍然只包含上层 LLM 指控系统、任务网关和 VBS Adapter，不实现下层 VBS 仿真引擎、行为树运行时和真实 socket/HTTP return。

但 Adapter 预览必须清楚展示上层将发给下层的结构化任务请求，包括：

- 我方班组 ID。
- 目标入口坐标。
- 最终路径 waypoints。
- 当前约束摘要。
- 行为树或 pending tactical intent。
- HTTP/socket 配置模式。

## 测试设计

必须覆盖以下验收点：

- 一键部署后地图上有我方班组、目标入口、敌方/风险对象和基础 2D 地图。
- 拖拽或点击部署能改变场景对象坐标。
- 默认寻路能生成至少 2 条候选路径，推荐结果可视化。
- 用户输入“大路有狙击风险”后，LLM 生成约束补丁，算法重规划并避开大路。
- 用户输入“敌军不在营地，必须争分夺秒”后，LLM 生成忽略敌区或时间优先补丁，算法重规划更短路径。
- 分数接近时触发 LLM 候选路径解释，并支持用户选择。
- 会话记忆能保存历史约束，重置会话能清空约束。
- GraphRAG 官方示例可跑通。
- 用户提供 PDF/DOCX 能建库、查询并返回来源。
- 最终 Adapter 预览与地图最终路径一致。

## 成功标准

用户打开系统后，不需要读源码即可完成以下演示：

1. 构建或加载 GraphRAG 知识库。
2. 启动 Web UI。
3. 一键部署默认场景。
4. 看到 wargame 风格 2D 地图和敌我目标对象。
5. 输入“前往目标建筑入口开展围剿”。
6. 看到 Top-N 路径候选。
7. 输入“大路有狙击风险”，看到路径改走小路。
8. 输入“敌军不在营地，必须争分夺秒”，看到路径重新偏向最短路。
9. 查看 LLM 解释、GraphRAG 命中来源、Adapter 请求预览。
10. 重置会话并从零开始演示。
